# 📝 学習記録（あなたの「考えた跡」を残す場所）

ここに各セッションの学習記録を保存します。
**「やったこと」だけでなく「自分で考えたこと」を残す**のが上達のコツです。

---

## ファイル命名規則

```
YYYY-MM-DD-session-XX-テーマ.md
```

例：
- `2026-04-13-session-00-project-overview.md`
- `2026-04-13-session-01-git-basics.md`

---

## 📋 記録テンプレート（コピペして使ってOK）

```markdown
# Session XX: テーマ

📅 日付: 2026-MM-DD
⏱ 所要時間: 約XX分
🎯 今日のゴール:

---

## 🧠 自分で考えてみた（先にメモ）

> セッション開始時のミニ課題に対して、Claude Code に聞く前に自分で考えた内容を書く欄。
> 当たっていなくてOK。「なぜそう思ったか」を残すのが上達の近道。

-

---

## 📚 今日学んだこと

-

---

## 💡 自分の言葉で説明してみる

> 学んだことを「初心者の自分」に説明するつもりで書く。書けないところは理解できていない部分。

-

---

## 🤔 つまずいたポイント＋どう抜けたか

> エラー・勘違い・「ここ何でこうなるの？」を残す。後で読み返すと、自分の成長が見える。

-

---

## ❓ まだよくわからないこと

-

---

## 🎯 次にやること

- [ ]
```

---

## 💪 上達する書き方のコツ

1. **「自分で考えてみた」を空欄にしない** — 5分でいいから、AI に聞く前に考える時間を作る
2. **「自分の言葉で説明してみる」を必ず書く** — メタ認知が一気に上がる
3. **「つまずいたポイント」は具体的に** — 「エラーが出た」ではなく「`useEffect` で無限ループした、原因は依存配列を空にし忘れた」
4. **完璧を目指さない** — 5分で書ける範囲でOK。書く習慣の方が大事

---

## 📊 一覧で振り返る

学習が進んだら Claude Code に：

```
knowledge/sessions/ の学習記録を全部読んで、
私の理解度の傾向（強み・弱み）を分析して
```

と聞くと、自分の学習傾向を客観視できます。
