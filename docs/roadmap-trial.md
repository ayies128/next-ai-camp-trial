## Session 1: Git ってなに？ — ファイルの変更を記録する方法

**プログラミングの世界では「ファイルの変更履歴を残す」のが基本ルールです。
そのためのツールが Git（ギット）です。**

このセッションで学ぶこと：
- Git って何？（「セーブポイント」を作るツールだと思ってください）
- なぜ Git が必要なの？（うっかり壊しても元に戻せる安心感）
- よく使うコマンド 4 つだけ覚える
  - `git status` — 今どんな状態？（変更があるか確認）
  - `git add` — 「これを記録するよ」と選ぶ
  - `git commit` — 記録する（セーブポイントを作る）
  - `git log` — 過去のセーブポイント一覧を見る

| やること | Claude Code への聞き方 |
|---------|----------------------|
| Git が何か理解する | 「Git って何？ゲームのセーブみたいなもの？」 |
| Git の初期設定をする | 「Git の初期設定をして。名前は〇〇、メールは〇〇で」 |
| 4 つのコマンドを試す | 「git status って何がわかるの？」 |
| 練習コミットを作ってみる | 「試しにコミットしてみたいんだけど、手順を教えて」 |

## Session 2: GitHub を使ってみよう — ブランチと PR

**ここでは、チーム開発の基本である「ブランチ」と「PR」を学びます。**

このセッションで学ぶこと：
- 自分のリポジトリに変更をプッシュ（アップロード）する方法

| やること | Claude Code への聞き方 |

## Session 3: オンボーディング — Web の世界を知ろう

**プログラミングの基礎に入る前に、Web の全体像を知るセッションです。**

このセッションで知ること：
- Web ページが表示される仕組み（URL を入力してからページが出るまで）
- Web を作る 3 つの言語の役割
  - **HTML** = ページの構造（骨組み）
  - **CSS** = 見た目（デザイン・色・配置）
  - **JavaScript** = 動き（ボタンを押したら何か起きる）
- ブラウザの「開発者ツール」の開き方（F12 キー）

| やること | Claude Code への聞き方 |
|---------|----------------------|
| Web の仕組みを知る | 「URL を入力してからページが表示されるまで、何が起きてるの？」 |
| 3 つの言語の違い | 「HTML、CSS、JavaScript は何が違うの？料理に例えて教えて」 |
| 開発者ツールを触る | 「ブラウザで F12 を押したんだけど、この画面の見方を教えて」 |
