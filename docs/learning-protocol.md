# 学習プロトコル（毎回の進め方）

> 1 セッション = 30〜60 分。6 つのフェーズを毎回実行します。すべて Claude Code で完結。

---

## 概要

毎回のセッションは以下の 6 フェーズで進めます：

```
Phase 1: 計画（5分）     → 今日のテーマを決める
Phase 2: 理解（15分）    → 教材で学ぶ
Phase 3: 実践（15-25分） → 手を動かす
Phase 4: 検証（5分）     → 理解度チェック
Phase 5: 記録（5分）     → 学習ログを残す
Phase 6: アウトプット（5分） → 学んだことを自分の言葉で伝える
```

---

## Phase 1: 計画（5分）

**目的**: 今日学ぶテーマを決める。

Claude Code を起動して、以下のプロンプトを入力してください：

> 📋 固定プロンプト: [01-theme-setting.md](../knowledge/prompts/01-theme-setting.md)

```
docs/roadmap.md と knowledge/sessions/ を読んで、
次に学ぶべきセッションのテーマを提案して。
30〜60分で完了する粒度で、「目的」「事前知識」「到達点」「ミニ課題」を含めて。
```

Claude Code がロードマップと過去の学習記録を自動で読み取って、最適なテーマを提案します。

---

## Phase 2: 理解（15分）

**目的**: 今日のテーマの全体像を掴む。

> 📋 固定プロンプト: [02-material-generation.md](../knowledge/prompts/02-material-generation.md)

```
今日のテーマは「{テーマ名}」です。
初心者向けの教材を作って。
構成: 理論説明 → コード例 → 実践課題（1〜2個）
60分以内で完了する分量で。
```

### フェーズ 0〜B（Session 0-14）の場合
- 概念説明を丁寧に読む
- わからない部分はすぐ Claude Code に質問
- コード例は **自分の手で写経**（コピペしない）

### フェーズ C 前半（Session 15-16）の場合
- DB 設計・HTTP・アプリ設計の**概念**を理解する
- 「読書管理アプリ」の例で具体的にイメージを掴む
- 自分のアプリの設計図（テーブル設計・画面一覧・機能一覧）を作る

### フェーズ C 後半（Session 17-20）の場合
- Session 15-16 で作った設計図をもとに、Claude Code と一緒に実装する
- コード例を参考に、設計パターンや実装方針を理解する

---

## Phase 3: 実践（15〜25分）

**目的**: 手を動かして体で覚える。

### 準備

```bash
# 新しいブランチを作成
git checkout -b feat/session-{番号}-{テーマ}
```

### フェーズ 0〜B（Session 0-14）

Claude Code に質問しながら、**自分の手でコードを書く**：

```
# わからないところを聞く
この for ループの i++ の部分がわからない。何をしてるの？

# 書いたコードを見てもらう
こう書いてみたけど合ってる？

# レビューしてもらう
今書いたコードをレビューして、改善点を教えて
```

### フェーズ C 前半（Session 15-16）

設計の考え方を学び、自分のアプリの設計図を作る：

```
# DB 設計を学ぶ
読書管理アプリに必要なテーブルを一緒に考えて

# 自分のアプリを設計する
自分のアプリに必要な画面・テーブル・機能の一覧を作って

# Claude Code への指示を練習
「本の一覧を表示する」を、具体的な指示に書き換えてみて
```

### フェーズ C 後半（Session 17-20）

Session 15-16 で作った設計図をもとに、Claude Code と一緒に実装：

```
# 設計図を参照して実装を依頼
Session 16 で作った設計図のテーブルを Supabase に作って

# 機能の実装を依頼
Supabase でユーザー認証機能を実装して。各ファイルの説明もつけて。

# 生成コードの理解を確認
今作ってくれたコードの中で、useEffect の部分を詳しく説明して
```

### コミット

作業の区切りごとに小さくコミットを積みましょう：

```
変更内容を見て、適切なコミットメッセージでコミットして
```

---

## Phase 4: 検証（5分）

**目的**: 今日学んだ内容が腹落ちしているか確認。

> 📋 固定プロンプト: [03-understanding-check.md](../knowledge/prompts/03-understanding-check.md)

```
今日の学習内容をもとに、理解度を確認する 3〜5 問の選択問題を作って。
答えと解説も最後につけて。
```

- 間違えた問題は Claude Code に追加で説明を求める
- 全問正解を目指す必要はなく、**理解を深めることが目的**

---

## Phase 5: 記録（5分）

**目的**: 学習記録を Markdown で作成して Obsidian に保存。

> 📋 固定プロンプト: [04-learning-summary.md](../knowledge/prompts/04-learning-summary.md)

```
今日のセッション内容をまとめて、学習記録を作成して
knowledge/sessions/ に保存して。
ファイル名は {YYYY-MM-DD}-session-{番号}-{テーマ}.md で。
```

Claude Code がセッションの内容を自動でまとめてファイルに保存します。
保存後、Obsidian で内容を確認・加筆してください。

---

## Phase 6: アウトプット（5分）

**目的**: 学んだことを自分の言葉で誰かに伝える。

> 📋 固定プロンプト: [05-output-formatting.md](../knowledge/prompts/05-output-formatting.md)

「人に説明する」ことで理解が深まります。方法は自由です：

| 方法 | やり方 |
|------|-------|
| **Claude Code に説明する** | 「今日学んだことを説明するから、合ってるか確認して」 |
| **X（Twitter）に投稿する** | 学習記録を要約して投稿。`#プログラミング学習` などのタグをつける |
| **ブログに書く** | Qiita、Zenn、note、はてなブログなど。Claude Code に記事整形を頼める |
| **友達・家族に話す** | 「今日こういうことを学んだんだけど…」と口頭で説明 |
| **メモに書く** | Obsidian のノートに「自分の言葉で」まとめ直す |

> 💡 一番手軽なのは Claude Code に説明すること。
> 「今日学んだ〇〇を自分の言葉で説明するね」と言って、合ってるか確認してもらいましょう。

---

## セッション・チェックリスト

毎回の終了時に確認：

- [ ] Phase 1: テーマを決めた
- [ ] Phase 2: 教材を読んで理解した
- [ ] Phase 3: 手を動かして実践した（ブランチ作成・コミット済み）
- [ ] Phase 4: 理解度チェックを実施した
- [ ] Phase 5: 学習記録を Obsidian に保存した
- [ ] Phase 6: 学んだことを自分の言葉でアウトプットした
- [ ] Git: **PR を作成した**（→ レビュー → マージ）

---

## Git フロー

### セッション開始時

```bash
# ブランチを作成して作業開始
git checkout -b feat/session-{番号}-{テーマ}
```

### 作業中

こまめにコミットを積みましょう（Claude Code に依頼してもOK）：

```
変更内容を見て、適切なコミットメッセージでコミットして
```

### セッション終了時 — PR を作成

毎セッションの終わりに PR を出します。Claude Code に依頼するだけ：

```
今日の変更で PR を作って。
タイトルは「Session #{番号}: {テーマ名}」で。
```

PR が作成されたら、内容を確認して **マージ** します：

```
この PR をマージして
```

### フェーズ終了時

フェーズの最後のセッションでは、振り返りも含めます：

```
このフェーズで学んだことを振り返って、次のフェーズに進む準備ができているか確認して
```

### ブランチ命名規則

```
feat/session-{番号}-{テーマ}
```

例: `feat/session-04-javascript-basics`

### PR テンプレート

```markdown
## 概要
- Session #{番号}: {テーマ名}
- フェーズ: {0 / A / B / C}

## 変更点
-

## 学んだこと
-

## 学習メモ
- `knowledge/sessions/{日付}-session-{番号}-{テーマ}.md`
```

## 前回の続きから始めたいとき

### 方法 1: Claude Code に「続き」と言う（おすすめ）

新しいセッションで Claude Code を開いて、こう言うだけで OK：

```
続き
```

Claude Code が `knowledge/sessions/` の学習記録を読んで、
前回どこまで進んだかを確認し、次のセッションを案内してくれます。

### 方法 2: 前回の会話をそのまま再開する

前回の会話のまま続けたい場合（ターミナル版）：

```bash
# 直前の会話を再開
claude -c

# 過去の会話一覧から選んで再開
claude -r
```

> VS Code 版では、Claude Code パネルに前回の会話が残っています。

---

## 困ったときは

```
# Claude Code の使い方がわからない
→ docs/claude-code-guide.md を読む

# 何を学べばいいかわからない
→ docs/roadmap.md を確認して Claude Code に相談

# エラーが出た
→ エラーメッセージをそのまま Claude Code に貼り付けて質問
```
